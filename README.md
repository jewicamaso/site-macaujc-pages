# site-macaujc-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何单个域名或具体网站，仅作为静态页面资料的整理与托管使用。

## 项目简介

这是一个以 HTML 页面为核心的归档仓库，主要服务于页面文件的版本管理与历史记录保存。仓库内的页面均为独立文件，彼此之间无强依赖关系，适合用于存放各类静态页面示例、原型、存档或临时展示内容。

## 目录说明

```
/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 页面归档目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源目录（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：存放主要 HTML 页面文件，每个文件独立，可单独访问。
- `assets/`：存放页面所需的公共资源文件，如样式表、JavaScript 脚本、图片等。
- 根目录下的 `index.html` 可作为仓库的导航或说明页，非必须。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可直接在浏览器中打开。
- 页面内引用的资源路径建议使用相对路径，以便在本地或 GitHub Pages 环境下正常访问。
- 仓库不保证页面内容的时效性或准确性，仅作为资料存档用途。
- 页面内容可能包含占位文本、示例数据或测试功能，不代表最终版本。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-macaujc-pages.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 如需部署到 GitHub Pages，请确保仓库设置中已启用 Pages 功能，并选择合适的分支与目录。
